## SOLUTION

---------------------------------------------

- Reference: https://portswigger.net/web-security/sql-injection
- SQL Injection Cheat sheet: https://portswigger.net/web-security/sql-injection/cheat-sheet

---------------------------------------------

Products are listed based on `category` parameter `/filter?category=Corporate+gifts`, which the backend uses in a `WHERE` clause.
The `category` value is concatenated directly into the SQL query, allowing you to appemd a `UNION SELECT` and have its results rendered with the product list.

>It can be determined that:
>  - The query returns 2 columns.
>  - Both columns accept and display text.
>
>PAYLOAD: `'+UNION+SELECT+'abc','def'--`


<img width="1728" height="982" alt="image" src="https://github.com/user-attachments/assets/99668554-93be-47c1-8b22-ab6241c517d0" />

-----------------------------------------------

As the lab description explicitly tells us that there is a `users` table with `username` and `password` columns.

PAYLOAD: `'+UNION+SELECT+username+password+FROM+users--`

<img width="1920" height="1152" alt="image" src="https://github.com/user-attachments/assets/dbdc742a-ed48-41f5-89dc-81f886d5a40c" />

Because the columns align (2 text fields), the response now includes each `username` and `password` pair as extra rows in the product listing.
